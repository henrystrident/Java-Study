哈希表通过哈希函数把键映射到桶，平均查询、插入、删除为 `O(1)`，最坏可退化。正确性还依赖 `equals` 与 `hashCode` 的一致性：相等对象必须有相同哈希值。

LRU 缓存需要同时做到 `O(1)` 查询与 `O(1)` 更新：哈希表定位节点，双向链表维护新旧顺序。访问节点后将其移到表头，容量超限时淘汰表尾节点。这是“组合数据结构”的经典范例。

练习：`LruCacheExercise`。

## LRU 是怎样协作的

假设容量为 2：依次 `put(1,A)`、`put(2,B)` 后，新旧顺序是 `2 → 1`；调用 `get(1)` 后变为 `1 → 2`；再放入键 3 时，键 2 位于尾部，因此被淘汰。

哈希表回答“节点在哪里”，双向链表回答“谁最新、谁最旧”。链表使用头尾哨兵后，插入头部和删除尾部都不需要为首尾节点写特殊分支。

## 必须维持的不变量

- 哈希表中的每个键都恰好对应链表中的一个节点。
- 越靠近头部表示越近期使用，尾部前一个节点最旧。
- 更新已有键不会增加缓存大小，但也要把节点移到头部。
- 删除节点时，哈希表和链表必须同时删除。

时间复杂度：`get`、`put` 平均为 `O(1)`；空间复杂度为 `O(capacity)`。
