综合题的难点通常不是某个新算法，而是识别模型：

- “先修关系、依赖顺序”通常对应有向图与拓扑排序。
- “最少次数、最少步数”常对应 BFS 或最短路径。
- “所有组合/排列”常对应回溯。
- “最大/最小值且选择互相影响”常尝试 DP 或贪心证明。

`CourseScheduleExercise` 将课程依赖建模为有向图，通过入度数组和队列判断能否完成，并给出一种合法学习顺序。

## 从题意到模型

若依赖对 `[course, prerequisite]` 为 `[1,0]`、`[2,0]`、`[3,1]`、`[3,2]`，应建立边 `0→1`、`0→2`、`1→3`、`2→3`。初始只有课程 0 入度为 0；完成它后课程 1、2 可学习；两者都完成后课程 3 的入度才降为 0。

Kahn 算法步骤：

1. 建立邻接表和每个节点的入度。
2. 把所有入度为 0 的节点加入队列。
3. 每取出一个节点，就把它指向节点的入度减一；新出现的 0 入度节点入队。
4. 若处理节点数小于课程总数，剩余部分存在环，因此无法完成。

## 综合检查清单

- 输入中的边方向是否建立正确？
- 孤立课程是否也计入结果？
- 重复依赖的处理规则是什么？
- 空依赖、单课程、自环和多节点环是否覆盖？
- 能否解释时间复杂度为什么是 `O(V + E)`？
